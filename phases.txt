NORTH STAR
====================================================

Tales of Drak is a custom-world, D&D 5e/SRD-inspired, Skyrim-style third-person fantasy sandbox RPG with:

- D&D 5e/SRD-style rules brain
- Skyrim-style third-person open-world exploration
- ARK-style gathering, crafting, stations, and building

- live action combat

- quests

====================================================
MASTER ARCHITECTURE RULE
====================================================

GameState is the single source of truth.

GameState owns:
- character name
- species/race
- class
- subclass
- ability scores later
- HP
- max HP
- XP
- level
- XP to next level
- inventory
- equipment
- item definitions
- crafting recipes
- action log
- last action result

HUD displays GameState.
HUD buttons call GameState actions.
WorldController handles only world interactions.
DungeonController handles only dungeon interactions.

No duplicated inventory mutation logic outside GameState.
No duplicated HP mutation logic outside GameState.
No duplicated XP mutation logic outside GameState.
No duplicated craft/use/equip logic outside GameState.

If a system breaks, fix the architecture instead of adding patches everywhere.

- starter_bandage
  - requirements: weathered_timber x2
  - output: starter_bandage x1

- camp_marker
  - requirements: weathered_timber x5, torch_kit x1
  - output: camp_marker x1

craft_recipe(recipe):
- accepts a recipe dictionary
- checks requirements
- if missing material, return failure and log exact reason:
  CRAFT failed: need Weathered Timber 0/2
- if successful:
  - remove materials
  - add output item
  - emit runtime_state_changed
  - log:
    CRAFT success: Simple Bandage -> Simple Bandage x1

use_item(item_id):
- check item exists
- check usable
- starter_bandage heals HP
- using bandage consumes 1 bandage
- refresh HP and inventory

equip_item(item_id):
- check item exists in inventory
- check equippable
- check valid slot
- set equipment[slot] = item_id
- emit runtime_state_changed
- log:
  EQUIP success: Rusty Sword -> main_hand

damage_player(amount, source):
- subtract HP
- clamp HP between 0 and max_hp
- emit runtime_state_changed
- log:
  DAMAGE enemy: 24 -> 19 (-5)

add_xp(amount):
- add XP
- support level up if XP >= xp_to_next
- emit runtime_state_changed
- log XP result

get_inventory_lines():
- return every inventory item with qty > 0
- chest loot and crafted items must appear immediately

get_character_view_lines():
- show:
  - character summary
  - HP
  - XP
  - AC
  - main hand
  - off hand
  - armor
  - weapon damage bonus

get_debug_lines():
- show:
  - GameState OK
  - current scene path
  - HP
  - XP
  - inventory dump
  - equipment dump
  - last action result
  - last 10 action log entries
