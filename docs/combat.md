# Combat

Combat is intentionally light at this stage.

For Framework Prototype v0.1, combat exists mainly as documentation and data scaffolding.

Current assumptions:

- combat will be class-informed
- equipment will influence stat outputs
- enemies will use simple role tags such as melee, ranged, elite, or boss
- dungeons should support co-op encounters later
- combat logic should remain separate from raw content data

Not yet implemented:

- ability system
- threat system
- status effects
- animation state handling
- encounter scripting
